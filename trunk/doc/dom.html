Package: DOM

Description

This is the root hierarchy of the DOM packages. DOM is a standard API to interface to XML trees. It is divided into several main parts (Core, Events, ...). Note that to be fully compliant, strings must be UTF-16 encoded. However, you still have the choice in this implementation, so that you can use UTF-8 encoding for documents that only have ASCII characters (and thus save some space in memory). This encoding is in the file xml/encodings.ads

The comments in these packages are not meant as a tutorial, or even a reference documentation for DOM. Please see the official XML documentation.
